CS110: Problem Solving with Data Structures and Algorithms
Spring 2026
Course Description
In this course, students learn how to apply core concepts in designing and analyzing algorithms and data
structures to address computational problems. Searching and sorting are a few examples of algorithmic strategies
students learn to solve problems ranging from logistics and route optimization to auto-completion engines and
plagiarism detection.
The CS110 course guide is an integral part of this syllabus and includes clarifications on the policies adopted in
this course. Please bookmark it on your browser and refer to it regularly during the semester, especially in
preparation for the work products to be submitted as assignments.
Course Objectives & Learning Outcomes
Analyze, evaluate, and compare algorithmic strategies and data structures to develop effective and efficient
solutions for computational problems, making informed decisions based on their strengths, limitations, and
computational complexity.
#cs110-AlgoStratDataStruct: Explain and apply the underlying principles of algorithmic techniques
(including sorting algorithms, randomization techniques, dynamic programming, and greedy algorithms,
among others), as well as data structures (their abstractions, implementations, and functional
operations).

a problem using the appropriate Big-O, Big-Ω, or Big-Θ notations.

Prerequisites & Working Knowledge
Completing PR51 (Spring Structured Study Sessions, S^4) is a prerequisite for this course, which can be done by
passing a technical interview or participating in the weekly sessions. The S^4 resources are listed on the SSS
myMinerva page, including the interview study guide materials and all weekly Python notebooks. Students are
expected to have a solid handle on this material for CS110, including variable scope, variable mutability, basics of
OOP, binary numbers (including float representation basics), operations on lists/tuples, and recursion, and should
take extra time to review certain topics as needed.
CS110 is not a course about Python Programming. It is a course about algorithms and data structures
implemented in Python. The course will assume a basic knowledge of significant data structures (such as lists and
dictionaries) and essential Python fluency at the summer preparatory materials level. To be prepared for the
course, please review sections 1 through 12 and 16-19 of "How to Think Like a Computer Scientist: Interactive
Edition." Retrieved from August 24, 2020, https://runestone.academy/ns/books/published/thinkcspy/index.html.
This reading will provide the necessary programming background to succeed in this course.
Make sure to do the interactive exercises contained in each section. Watching the videos contained therein is
not mandatory but highly recommended.
After reading this resource, you should be able to:
Use built-in Python data structure methods with dictionaries and lists,
Implement essential Python functions, such as reversing lists or generating random numbers.


Import known Python modules, such as random and NumPy.
For students who have mastered the skills above, please familiarize yourself with Python classes—this blog
post is particularly instructive, and you should go through it before the course starts since we will cover it
roughly halfway through the semester.
For students looking for more resources, this document contains helpful tips.
Note: Please submit code that is at least compatible with Python 3.6, and avoid using new functionalities provided
by more recent versions unnecessarily.
Assignments
Please refer to the CS110 course guide for all the details on submitting your CS110 assignments in Forum and
what is expected from your submission.
All assignments are strictly individual. Note the weighting of each assignment:
Projects 1 and 2 each represent about 3.5 weeks of coursework. They assess all six LOs and one HC. They
give you the opportunity to demonstrate progress across the full skills taught and practised in class. Their
weighting reflects where your consistent time and effort will have the most significant impact and how the
work you do in and out of class builds the independence to tackle new and complex problems, which is one
of the main goals of this course.
Pseudo Project 2 is narrower in scope. It assesses one technical LO and one HC. Its higher relative weight of
2x compared to 1x for the other projects is intentional. This assignment provides a high-impact chance to
plan, experiment, and grow before the final project. You will also receive targeted feedback on your
approach. The impact on the overall course grade is about half that of a whole project, so you can give it the
focus it deserves.

Unit 1: Algorithms—Design and Analysis
How do we choose an optimal algorithm for a task when multiple choices exist? What does it mean to analyze an
algorithm? What properties would we want to measure? We begin discussing these questions in this unit and start
rigorously answering them in the context of basic sorting algorithms.


We explore one of the most crucial algorithmic design paradigms, divide-and-conquer, and provide tools for
analyzing algorithms of this type, along with asymptotic complexity analysis. Merge sort, introduced in Formal
Analyses, and solutions to the maximum subarray problem are illustrative examples.
This unit provides critical background knowledge for CS162 (Software Development: Developing Powerful
Applications) and CS142 (Computability and Complexity).

Unit 4: Greedy Techniques and Dynamic Programming
Dynamic programming is another algorithmic design paradigm studied in this course. Like divide-and-conquer, it is
also based on breaking down problems into smaller components. We learn to use dynamic programming by
analyzing and seeking optimal solutions to a rod-cutting problem and the longest common subsequence.
We also cover the greedy algorithm design paradigm, first encountered in Formal Analyses, and implement
Huffman codes.
This unit provides background for CS162 (Software Development: Developing Powerful Applications), CS152
(Harnessing Artificial Intelligence Algorithms), and CS156 (Machine Learning for Science and Profit).

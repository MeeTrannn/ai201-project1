CS162: Software Development: Building Powerful Applications
Spring 2026

Course Description
This course is organized around the principle that the only way to learn software development is to
develop software. Collaborate as a team to develop a substantial web application. Examples include, but
are not limited to, social media applications or distributed chat systems. You will have the opportunity to
apply and experience all aspects of software development, including requirements analysis, design,
planning, implementation, testing, and deployment.
The Conquering CS162 document is an integral part of this syllabus and includes clarifications on the
policies adopted in this course. Please bookmark it on your browser and refer to it regularly during the
semester, especially in preparation for the work products to be submitted as assignments.

Unit 5: Testing, Deployment, and Security
As systems grow it becomes increasingly difficult to change anything without breaking something. By
writing comprehensive tests for the code, it becomes possible to introduce significant changes while
remaining confident that any new bugs or regressions will be quickly discovered by the test suite.
Large projects very often decompose into several subsystems that need to work together collectively to
function correctly. Sometimes changing one subsystem breaks the overall system, so integration testing
is used to make sure that all subsystems are currently working well together. This whole process can be
automated into a process known as continuous integration.
Although the system or application gets developed on a local machine, it still needs to be deployed onto
a remote server. A poor deployment process will have many inconsistencies between the two
environments, take the production system offline while the deployment is taking place and will require
extensive manual intervention to get the system working again. This can significantly reduce the ability to
scale out the application. Various best practices can help avoid the worst problems.

Makeup Work
For this course, the makeup work process is as follows:
1. Student misses class.


2. The student has 7 days from the end of class to submit their PCW by committing the work to their
repo. (If there are multiple successive absences, then the deadline for all PCW will be 7 days after
the last absence.)
3. They must email either their TA or Prof to review the PCW.
4. Once the PCW has been reviewed then the process is as follows:
1. If the MUW is insufficient then we will set an additional score from a 0 to a 1.
2. A comment will mention that your work was submitted, and was adequate. In some cases,
everyone's PCW is graded, and then your PCW will (probably) be graded too.
